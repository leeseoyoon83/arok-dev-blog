# 제품 요구사항 정의서 (PRD - Product Requirements Document)

본 문서는 성장과 자기계발을 지향하는 블로그 플랫폼 **"멘토의 공간"**의 기획 및 개발 사양을 정의합니다.

---

## 1. 프로젝트 개요

* **서비스명**: 멘토의 공간 (Mentor's Space)
* **컨셉**: 모던하고 따뜻한 멘토의 감성이 느껴지는 감성 블로그
* **주요 대상**: 성장과 자기계발, 독서, 기록에 관심이 높은 주니어 개발자 및 학습자들
* **핵심 가치**: 배움의 나눔, 지식 자산화, 마음을 가꾸는 멘토링 아카이브

---

## 2. 기술 스택 (Technical Stack)

본 프로젝트는 빠르고 유연한 개발과 안정적인 데이터 보존을 위해 아래의 모던 기술 생태계를 기반으로 구축되었습니다.

* **Frontend Framework**: `React (v18+)`
  * 컴포넌트 기반 설계를 통한 UI 재사용성 극대화 및 선언적 상태 관리(`useState`, `useEffect`) 적용
* **Build & Dev Tool**: `Vite (v5+)`
  * 초고속 HMR(Hot Module Replacement) 지원으로 쾌적한 로컬 개발 속도 제공 및 최적화된 프로덕션 에셋 번들링 수행
* **Language**: `JavaScript`
  * 최신 ES6+ 사양의 자바스크립트를 활용한 동적 비동기 로직 및 컴포넌트 처리
* **Styling**: `Tailwind CSS` & `Vanilla CSS`
  * 여백 중심의 둥근 모서리 디자인, 부드러운 다크/라이트 테마 스위처 구현 및 유연한 유틸리티 스타일링 레이아웃 제공
* **Backend & Database**: `Supabase` / `LocalStorage` (Hybrid)
  * 클라우드 관계형 데이터베이스(PostgreSQL) 연동을 통한 글 CRUD 수행
  * 데이터베이스 미설정 환경에서도 중단 없는 로컬 테스트를 지원하기 위해 브라우저의 `LocalStorage`로 자동 복구되는 폴백 설계 제공

## 3. 기능 요구사항 (Functional Requirements)

### 3.1. 포스트 데이터 관리 및 조회
* **데이터 필드 스펙**: `id`, `title`, `content`, `category`, `tags`, `createdAt`
* **카테고리 구성**: 
  1. `멘토의 지혜` (성장 팁, 인사이트, 리더십 조언)
  2. `멘토의 책방` (인생 도서 추천 및 독서 가이드)
  3. `성장일기` (일상 회고 및 멘토의 개인 기록)
* **초기 데이터**: 요구 조건에 매치되는 5종의 더미 데이터 수록

### 3.2. 탐색 및 필터 기능
* **카테고리별 필터**: 상단 탭을 눌러 해당 카테고리의 글들만 그리드로 필터링하여 출력
* **태그 필터**: 글 카드에 노출된 해시태그를 클릭하면 동일 태그를 가진 포스트만 필터링
* **통합 검색**: 제목, 내용, 태그를 탐색하는 실시간 키워드 검색창 제공
* **필터 초기화**: 필터링 결과가 비어 있거나 필터를 해제하고 싶을 때 단번에 초기 상태로 리셋하는 기능

### 3.3. 회원 인증 및 관리자 모드
* **모의 로그인**: `admin123` 임시 비밀번호를 통해 즉석 관리자 계정 로그인 지원
* **상태 관리**: 새로고침 시에도 브라우저 내에 로그인 상태가 유지되도록 로컬 저장소 동기화
* **관리자 기능 활성화**: 관리자 로그인 시 일반 유저에게 숨겨져 있는 글쓰기 및 삭제 기능 오픈

### 3.4. 글 작성 및 삭제 기능 (관리자 전용)
* **글 작성 폼**: 제목, 카테고리(선택형), 태그(콤마 구분), 내용 입력기 제공
* **자동 문단 정제**: 단순 텍스트 개행을 웹 표준 문단 태그(`<p>`) 형태로 파싱하여 출력
* **글 삭제**: 글 카드 내 쓰레기통 버튼을 눌러 영구 삭제 처리 (이벤트 전파 방지 처리 적용)

### 3.5. 상세 읽기 모달 뷰
* **슬라이드 오버**: 카드를 클릭했을 때 나타나는 우측 슬라이드형 모달 상세 창
* **독서 프로그레스 바**: 모달 본문의 스크롤 정도를 모니터링하여 상단에 스크롤 높이를 백분율 너비로 시각화

## 4. 디자인 시스템 요구사항 (Design System)

* **에스테틱 컬러 팔레트 (Warm Beige & Brown)**:
  * 라이트 모드: 베이지 계열 배경(`#FAF6EE`), 다크 에스프레소 브라운 글자색(`#3E2723`), 카라멜 액센트 브라운(`#8D6E63`)
  * 다크 모드: 짙은 밤색 계열 배경(`#150F0D`), 소프트 코코아 글자색(`#F5EBE6`)
* **레이아웃**: 20px 이상의 시원하고 넓은 마진/패딩 여백 구성
* **카드 UI**: 둥근 모서리(`border-radius: 20px` 이상)와 가벼운 음영 효과를 통해 종이 잡지 느낌 표현
